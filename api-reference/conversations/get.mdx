---
title: 'Get Conversation'
api: 'GET /v1/conversation/:id'
description: 'Get details of a specific conversation'
---

## Endpoint

```
GET /v1/conversation/:id
```

## Authentication

Requires user token with conversation access or API key with `read:conversations` scope.

## Path Parameters

<ParamField path="id" type="string" required>
  Conversation ID
</ParamField>

## Example Request

```bash
curl -X GET \
  -H "x-api-key: Bearer YOUR_TOKEN" \
  https://api.maneged.com/v1/conversation/conv-123
```
