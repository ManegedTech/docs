---
title: 'Send Message'
api: 'POST /v1/message/send'
description: 'Send a message in a conversation'
---

## Endpoint

```
POST /v1/message/send
```

## Authentication

Requires user token or API key with `write:messages` scope.

## Request Body

<ParamField body="conversation" type="string" required>
  Conversation ID
</ParamField>

<ParamField body="message" type="string" required>
  Message content
</ParamField>

<ParamField body="reply_to" type="string">
  ID of message being replied to
</ParamField>

<ParamField body="attachments" type="array">
  Array of attachment objects with url and type
</ParamField>

## Example Request

<CodeGroup>
```bash Simple Message
curl -X POST \
  -H "x-api-key: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "conversation": "conv-123",
    "message": "Hello there!"
  }' \
  https://api.maneged.com/v1/message/send
```

```javascript With Reply
const response = await fetch('https://api.maneged.com/v1/message/send', {
  method: 'POST',
  headers: {
    'x-api-key': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    conversation: 'conv-123',
    message: 'Replying to your question',
    reply_to: 'msg-456'
  })
});
```
</CodeGroup>

## Response

```json
{
  "status": 201,
  "success": true,
  "data": {
    "$id": "msg-789",
    "conversation": "conv-123",
    "message": "Hello there!",
    "sender": "user-123",
    "created_at": "2025-01-19T10:00:00.000Z",
    "status": "SENT"
  }
}
```

<Note>
  Message is automatically sent to all participants via WebSocket.
</Note>
